# Analytics Agent

The Analytics Agent is a purely deterministic node. It does **not** use an LLM. Its sole purpose is to securely execute complex mathematical calculations using Pandas and NumPy, isolating the LLM from any arithmetic hallucination risk.

## Responsibilities
- Consumes the `analysis_plan` generated by the Planner.
- Performs GroupBys, standard deviations, correlations, and trend aggregations.
- Formats the resulting DataFrames into nested JSON structures.

## Internal Modules
- `statistical_engine.py`: Vectorized statistical functions.
- `trend_analyzer.py`: Time-series grouping and rolling averages.

## I/O Contract
**Reads from State:**
- `state["workflow_tracking"]["analysis_plan"]`
- `state["dataset_info"]["raw_dataframe"]`

**Writes to State:**
- `state["business_analytics"]`

## Time Complexity
Calculations are heavily vectorized utilizing underlying C-bindings in Pandas. Expected complexity for aggregations is `O(N log N)` for sorting and `O(N)` for summing.

## Failure Handling
Any Pandas `KeyError` or `ValueError` (e.g., attempting a sum on a string column mistakenly planned by the Planner) is caught. The agent logs the error and gracefully skips that specific calculation, continuing with the rest of the plan.
